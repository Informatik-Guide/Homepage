Eindimensional Datenstruktur
In diesem Tutorial werden wir wichtige Fachbegriffe der Programmierung erklären, damit du sie verstehen kannst, selbst wenn du noch keine Vorkenntnisse hast.
1. Referenz
Eine Referenz ist wie ein Zeiger oder eine Adresse, die auf ein Objekt zeigt. Stell dir vor, du hast ein Haus (das Objekt) und eine Karte mit der Adresse (die Referenz). Die Karte zeigt dir den Weg zum Haus, damit du darauf zugreifen kannst.

2. Kardinalität
Kardinalität beschreibt die Anzahl oder Beziehung zwischen Dingen. Zum Beispiel kann die Kardinalität zwischen einem Lehrer und den Schülern "eins zu viele" sein, was bedeutet, dass ein Lehrer viele Schüler hat, aber jeder Schüler nur einen Lehrer.
3. Array/Feld
Ein Array oder Feld ist wie ein Regal mit vielen Fächern, in denen Daten gespeichert werden können. Alle Fächer im Regal enthalten ähnliche Dinge, z. B. Zahlen oder Texte. Jedes Fach hat eine Nummer (Index), mit der man darauf zugreifen kann.

Ein Index ist wie eine Nummer, die angibt, wo sich etwas in einer Liste oder einem Array befindet. Die Nummerierung beginnt in den meisten Programmiersprachen bei 0. Wenn du zum Beispiel das dritte Element in einer Liste suchst, ist der Index 2.
5. Datenkapselung
Datenkapselung bedeutet, dass Daten vor direktem Zugriff geschützt sind. Man kann nur auf Daten zugreifen oder sie ändern, indem man spezielle Methoden verwendet. Dadurch wird die Integrität der Daten gewährleistet und unerlaubte Änderungen werden vermieden.

Zusammenfassung
In diesem Tutorial haben wir grundlegende Fachbegriffe der Programmierung erklärt:
• Referenz: Ein Verweis auf ein Objekt im Speicher.
• Kardinalität: Die Anzahl der Elemente oder die Beziehung zwischen Objekten.
• Array/Feld: Eine geordnete Sammlung von Elementen desselben Datentyps.
• Index: Eine Position, die verwendet wird, um auf ein Element zuzugreifen.
• Datenkapselung: Das Konzept, interne Details eines Objekts vor direktem Zugriff zu schützen.
